Análisis narrativo del Titanic: datos, limpieza y modelos en Python¶

Prerequisito recomendado: si aún no dominas CSV, Pandas y NumPy, empieza con fundamentos_csv_pandas_numpy.ipynb.

Este notebook usa el dataset clásico del Titanic para practicar un flujo completo de trabajo con datos:

  • cargar y entender una tabla,
  • detectar problemas de calidad,
  • limpiar valores faltantes y tipos incorrectos,
  • crear variables nuevas a partir de la información existente,
  • explorar patrones con tablas y gráficas,
  • construir un modelo logístico para estimar probabilidad de supervivencia.

La idea no es predecir con precisión perfecta, sino aprender a manejar información real: incompleta, desordenada y con decisiones de limpieza que afectan el análisis.

1. Carga de datos¶

Cada fila representa un pasajero del Titanic. Las columnas más importantes son:

  • Survived: si sobrevivió (1) o no (0).
  • Pclass: clase del ticket (1, 2 o 3).
  • Sex: sexo del pasajero.
  • Age: edad en años.
  • SibSp: hermanos o cónyuge a bordo.
  • Parch: padres o hijos a bordo.
  • Fare: precio del boleto.
  • Embarked: puerto de embarque (C, Q o S).
  • Cabin: número de cabina (muchos valores faltantes).
  • Name: nombre completo del pasajero.

Usaremos un CSV local en data/titanic.csv. Si no existe, intentamos descargarlo desde una URL pública.

3. Auditoría de calidad¶

La limpieza empieza con un diagnóstico. Revisamos:

  1. Duplicados: filas repetidas que podrían inflar conteos.
  2. Valores imposibles: edades negativas, tarifas negativas, clases fuera de rango.
  3. Cardinalidad: cuántos valores distintos tiene cada columna categórica.
  4. Consistencia: si los tipos de datos coinciden con el significado de la columna.

Esta auditoría nos dice qué hay que corregir y por qué.

4. Limpieza de datos¶

Vamos a limpiar paso a paso y documentar cada decisión.

Decisiones de limpieza¶

  1. Embarked: solo faltan 2 valores. Los reemplazamos por la moda (S), el puerto más frecuente.
  2. Age: faltan 177 edades. En lugar de borrar esas filas, imputamos la mediana por Pclass y Sex, porque la edad depende mucho de la clase y del sexo.
  3. Fare: si hubiera nulos, usaríamos la mediana por clase. Aquí no hay nulos, pero sí revisamos valores extremos.
  4. Cabin: tiene demasiados faltantes para rellenar texto. Creamos has_cabin (1 si había cabina registrada, 0 si no).
  5. Tipos: convertimos variables categóricas a tipo category para ahorrar memoria y facilitar análisis.

Idea clave del curso: no existe una sola forma "correcta" de limpiar. Cada decisión cambia el análisis. Por eso documentamos el criterio.

In [8]:
print("--- Iniciando limpieza de datos ---")

df = df_original.copy()
filas_inicio = len(df)

# 1) Embarked: reemplazar nulos con la moda.
embarked_mode = df["Embarked"].mode(dropna=True)[0]
df["Embarked"] = df["Embarked"].fillna(embarked_mode)

# 2) Age: imputar mediana por Pclass + Sex.
age_median = df.groupby(["Pclass", "Sex"])["Age"].transform("median")
df["Age"] = df["Age"].fillna(age_median)

# 3) Fare: por si aparece algún nulo en otra versión del dataset.
fare_median = df.groupby("Pclass")["Fare"].transform("median")
df["Fare"] = df["Fare"].fillna(fare_median)

# 4) Cabin: crear indicador de si había cabina registrada.
df["has_cabin"] = df["Cabin"].notna().astype(int)

# 5) Tipos categóricos.
df["Sex"] = df["Sex"].astype("category")
df["Embarked"] = df["Embarked"].astype("category")
df["Pclass"] = df["Pclass"].astype("category")

# 6) Filtros de valores imposibles.
df = df[df["Age"] >= 0]
df = df[df["Fare"] >= 0]
df = df[df["Pclass"].isin([1, 2, 3])]

print(f"Filas al inicio: {filas_inicio}")
print(f"Filas después de limpieza: {len(df)}")
print(f"Filas eliminadas: {filas_inicio - len(df)}")

print("\nValores faltantes después de limpiar:")
display(df.isnull().sum().to_frame("faltantes"))

5. Creación de variables nuevas¶

Muchas veces la información útil no viene lista en una sola columna. Hay que construirla.

Crearemos:

  • family_size: tamaño del grupo familiar a bordo (SibSp + Parch + 1).
  • is_alone: 1 si viajaba solo, 0 si iba acompañado.
  • title: título extraído del nombre (Mr, Mrs, Miss, etc.).
  • age_group: rango de edad (child, young_adult, adult, senior).
  • fare_per_person: tarifa dividida entre el tamaño del grupo.
  • is_child: 1 si tiene menos de 18 años.

Estas variables ayudan a responder preguntas como:

  • ¿Viajar solo cambió la probabilidad de sobrevivir?
  • ¿Los niños tuvieron más probabilidades de ser rescatados?
  • ¿El título social se relaciona con la supervivencia?
In [9]:
# Tamaño del grupo familiar.
df["family_size"] = df["SibSp"] + df["Parch"] + 1

# Indicador de viaje solo.
df["is_alone"] = (df["family_size"] == 1).astype(int)

# Título a partir del nombre: texto entre coma y punto.
df["title"] = df["Name"].str.extract(r",\s*([^\.]+)\.", expand=False).str.strip()

# Agrupar títulos poco frecuentes en 'Other'.
title_counts = df["title"].value_counts()
rare_titles = title_counts[title_counts < 10].index
df["title"] = df["title"].replace(rare_titles, "Other")

# Grupos de edad.
bins = [0, 12, 25, 60, 120]
labels = ["child", "young_adult", "adult", "senior"]
df["age_group"] = pd.cut(df["Age"], bins=bins, labels=labels, right=False)

# Tarifa por persona del grupo.
df["fare_per_person"] = df["Fare"] / df["family_size"]

# Indicador de menor de edad.
df["is_child"] = (df["Age"] < 18).astype(int)

Cómo leer estas visualizaciones¶

  • Sexo: en el Titanic, las mujeres tuvieron una tasa de supervivencia mucho mayor. Esto refleja la política de evacuación "mujeres y niños primero".
  • Clase: a mayor clase (1 = primera), mayor supervivencia. La clase socioeconómica importó para el acceso a botes salvavidas.
  • Edad: los niños tuvieron mejores probabilidades relativas que muchos adultos.
  • Tarifa: quienes pagaron más suelen haber sobrevivido más, pero la tarifa también está ligada a la clase.
  • Correlaciones: Survived se relaciona con Sex codificado, Pclass, Fare y variables derivadas. No confundas correlación con causalidad.

7. Regresión logística de supervivencia¶

Después de limpiar y crear variables, podemos modelar la pregunta central:

¿Qué factores se asocian con la probabilidad de sobrevivir?

Usamos regresión logística porque Survived es binaria (0 o 1).

Variables del modelo:

  • Pclass
  • Sex
  • Age
  • Fare
  • family_size
  • is_child
  • has_cabin

La fórmula con statsmodels nos permite interpretar coeficientes de forma parecida al notebook de nutrición.

In [11]:
print("--- Regresión logística de supervivencia ---")

modelo_df = df[[
    "Survived", "Pclass", "Sex", "Age", "Fare",
    "family_size", "is_child", "has_cabin"
]].copy()

modelo_df["Sex"] = modelo_df["Sex"].astype(str)

formula = "Survived ~ C(Pclass) + C(Sex) + Age + Fare + family_size + is_child + has_cabin"
modelo_logit = smf.logit(formula, data=modelo_df).fit(disp=0)

display(modelo_logit.summary().tables[1])
print("Pseudo R-cuadrado:", round(modelo_logit.prsquared, 4))

Interpretación del modelo logístico¶

En regresión logística, un coeficiente positivo indica que aumenta la probabilidad del evento (sobrevivir). Un coeficiente negativo indica lo contrario.

Ejemplos típicos en este dataset:

  • C(Sex)[T.male] negativo → ser hombre se asocia con menor probabilidad de sobrevivir, comparado con mujeres.
  • Coeficientes de C(Pclass) → a peor clase, menor probabilidad relativa de supervivencia.
  • is_child positivo → los menores de edad tuvieron mayor probabilidad de ser rescatados.
  • has_cabin positivo → tener cabina registrada puede reflejar mayor estatus o ubicación favorable en el barco.

El Pseudo R-cuadrado da una idea general del ajuste, pero no se interpreta igual que en regresión lineal.

8. Evaluación con train/test split¶

Para revisar si el modelo generaliza, separamos datos de entrenamiento y prueba.

Esto es importante en cualquier curso de datos: no basta con ajustar bien en la misma muestra; hay que probar con datos que el modelo no vio durante el entrenamiento.

9. Resumen final: qué aprendimos sobre manejo de datos¶

En este notebook practicamos un flujo completo:

  1. Carga del dataset y primera inspección.
  2. Auditoría de calidad: nulos, duplicados, valores imposibles.
  3. Limpieza con criterios explícitos (moda, mediana por grupo, variables derivadas).
  4. Ingeniería de variables para capturar familia, edad, estatus y tarifa por persona.
  5. Exploración con tablas y gráficas para entender patrones.
  6. Modelado logístico para cuantificar asociaciones con la supervivencia.
  7. Evaluación con partición train/test.

Mensajes clave para el curso¶

  • Los datos rara vez llegan listos: la limpieza es parte del análisis, no un paso menor.
  • Cada decisión de imputación cambia resultados: conviene documentar y justificar.
  • Las visualizaciones ayudan a detectar sesgos y relaciones antes de modelar.
  • Un modelo puede ajustar bien, pero hay que interpretar con contexto histórico y ético.

La parte más importante no es solo correr el modelo, sino saber leer la historia que cuentan los datos.
